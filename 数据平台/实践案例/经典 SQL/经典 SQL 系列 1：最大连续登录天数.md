在用户行为分析中，最大连续登录天数是一个经典的指标，能直接反映用户的粘性与活跃度。无论是面试还是实际工作，这个问题出现的频率都非常高。本文将带你一步步拆解这个需求，通过 Hive SQL 的窗口函数，优雅地解决“连续区间”难题。

## 1. 数据准备

有一张用户登录日志表 user_login，记录用户每次登录的日期：
```sql
CREATE  TABLE IF NOT EXISTS user_login(
  uid string,
  login_time string
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LINES TERMINATED BY '\n';
```
我们需要计算每个用户的最大连续登录天数（按自然日连续），如果中间有一天未登录，连续性就中断。

## 2. 分析思路

解决连续问题的通用技巧是：通过排序编号与日期运算，为连续区间生成一个唯一的分组键。

### 2.1 生成行号

用 ROW_NUMBER() 按用户分组、按日期排序，为每个用户的登录日期生成一个从 1 开始连续递增的编号 rn。

### 2.2 计算基准日期

这是最关键的一步：用登录日期 login_time 减去 rn 天，得到基准日期(作为分组ID)：
- 如果登录日期是连续的，那么 login_time 和 rn 会同步递增，差值保持不变。
- 一旦日期出现中断，rn 依然会递增，但 login_time 的跳跃大于 1 天，导致差值发生变化，从而产生新的分组ID。

举个例子：用户 a 在 5月1日、5月2日、5月3日登录，rn 分别为 1、2、3：
```
date_sub('2026-05-01', 0) = '2026-05-01'
date_sub('2026-05-02', 1) = '2026-05-01'
date_sub('2026-05-03', 2) = '2026-05-01'
```
它们得到同一个基准日期 2026-05-01。如果 5月5日又登录，此时 rn=4：
```
date_sub('2026-05-05', 3) = '2026-05-02'，
```
基准日期变成了 2026-05-02，与前一组不同。这样我们就自动将连续区间划分开了。

> 详细请参考[Hive SQL 中的断点分组技术详解](https://smartsi.blog.csdn.net/article/details/146066793)

### 2.3 分组统计连续天数

按照 uid 和刚才算出的分组ID，`COUNT(*)` 就是该段连续登录的天数。

### 2.4 求最大值

最后，再按 uid 分组，取出最大连续天数即可。

```sql
SELECT
  uid, group_id,
  MIN(login_time) AS min_login_time,
  MAX(login_time) AS max_login_time,
  COUNT(*) AS days
FROM (
  SELECT uid, login_time, date_sub(login_time, idx) AS group_id
  FROM (
    SELECT
      uid, login_time,
      ROW_NUMBER() OVER(PARTITION BY uid ORDER BY login_time) AS idx
    FROM user_login
  ) AS a
) AS b
GROUP BY uid, group_id;
```

```sql
SELECT
  uid, min_login_time, max_login_time, days
FROM (
  SELECT
    uid, min_login_time, max_login_time, days,
    ROW_NUMBER() OVER(PARTITION BY uid ORDER BY days DESC) AS idx
  FROM (
    SELECT
      uid, date_sub(login_time, idx) AS group_id,
      MIN(login_time) AS min_login_time,
      MAX(login_time) AS max_login_time,
      COUNT(*) AS days
    FROM (
      SELECT
        uid, login_time,
        ROW_NUMBER() OVER(PARTITION BY uid ORDER BY login_time) AS idx
      FROM user_login
    ) AS a
    GROUP BY uid, date_sub(login_time, idx)
  ) AS b
) AS c
WHERE idx = 1;
```

## 4. 总结

解决连续登录天数这类问题的精髓在于：利用有序的行号，构造出能够标识连续区间的“锚点”。这个技巧不仅适用于登录场景，也适用于考勤打卡、活跃天数、连续下单等任何需要判断“连续”行为的分析。

掌握 ROW_NUMBER() + 日期偏移 的组合用法，你就能轻松应对一大类 Hive SQL 面试题，并在真实业务分析中游刃有余。

# 文件上传群组功能完善

## 设计思路

采用**两步方案**：
1. **上传阶段**：用户选择文件 → 前端 FileReader 本地预览 → 点"上传文件"上传到 MinIO → 上传成功后启用"保存群组"按钮
2. **保存阶段**：点"保存群组" → 后端从 MinIO 读取 CSV → 解析去重 → 写入引擎表 → 创建群组记录 → 返回导入统计

> **关键交互约束**：只有文件成功上传到服务器后，"保存群组"按钮才可点击。

## 现状问题

1. **核心功能缺失**: CSV 上传到 MinIO 后，`create()` 中仅有 TODO 注释，entity_id 永远不会写入 ClickHouse
2. **GroupTask 不支持 upload 类型**: `executeGroupSelection()` 只处理 rule 类型
3. **无数据预览/导入统计**
4. **模板列名不一致**: 模板用 `unique_id`，预览栏写 `entity_id`

## Task 3: 后端 - CSV 解析写入 ClickHouse 核心方法

**文件**: `profile-web/src/main/java/com/data/profile/web/service/GroupService.java`

新增 `importCsvToClickHouse()` 方法（public，供 GroupTask 复用）：

```java
/**
* 解析 MinIO 中的 CSV 文件，去重后写入 ClickHouse 群组表
*/
public Map<String, Object> importCsvToClickHouse(String groupId, String fileKey, String mode) {
    String resultTable = "profile_group_" + groupId;

    // 1. 从 MinIO 读取 CSV，解析+去重
    Set<String> entityIds = new LinkedHashSet<>();
    int totalRows = 0;
    int invalidRows = 0;
    try (InputStream is = minioService.getFileAsStream(fileKey);
         BufferedReader reader = new BufferedReader(new InputStreamReader(is, StandardCharsets.UTF_8))) {
        reader.readLine(); // 跳过表头
        String line;
        while ((line = reader.readLine()) != null) {
            totalRows++;
            String id = line.trim().split(",")[0].trim().replaceAll("^\"|\"$", "");
            if (id.isEmpty()) {
                invalidRows++;
            } else {
                entityIds.add(id);
            }
        }
    } catch (Exception e) {
        throw new RuntimeException("CSV 文件解析失败: " + e.getMessage());
    }

    // 2. 建表（如不存在）
    String createSql = String.format(
        "CREATE TABLE IF NOT EXISTS %s (" +
        "entity_id String COMMENT '实体ID', " +
        "_created_time DateTime DEFAULT now() COMMENT '导入时间'" +
        ") ENGINE = MergeTree() ORDER BY entity_id SETTINGS index_granularity = 8192",
        resultTable);
    analysisEngineService.executeStatement(createSql);

    if ("replace".equals(mode)) {
        analysisEngineService.executeStatement("TRUNCATE TABLE " + resultTable);
    }

    // 3. 批量 INSERT（每 1000 条一批）
    List<String> batch = new ArrayList<>(1000);
    for (String entityId : entityIds) {
        batch.add("('" + entityId.replace("'", "\\'") + "')");
        if (batch.size() >= 1000) {
            analysisEngineService.executeStatement(
                "INSERT INTO " + resultTable + " (entity_id) VALUES " + String.join(",", batch));
            batch.clear();
        }
    }
    if (!batch.isEmpty()) {
        analysisEngineService.executeStatement(
            "INSERT INTO " + resultTable + " (entity_id) VALUES " + String.join(",", batch));
    }

    // 4. 统计
    Map<String, Object> stats = new HashMap<>();
    stats.put("total_rows", totalRows);
    stats.put("valid_ids", totalRows - invalidRows);
    stats.put("duplicate_ids", totalRows - invalidRows - entityIds.size());
    stats.put("imported_count", entityIds.size());
    return stats;
}
```

## Task 5: 后端 - GroupTask 支持 upload 类型

**文件**: `profile-web/src/main/java/com/data/profile/web/task/GroupTask.java`

`executeGroupSelection()` L95 的判断改为分支：

```java
if (groupRule == null) {
    throw new IllegalStateException("群组 " + groupId + " 规则为空");
}

if ("rule".equals(groupRule.getType())) {
    // 现有规则翻译逻辑（保持不变）...
} else if ("upload".equals(groupRule.getType())) {
    String fileKey = groupRule.getUuidFileKey();
    if (StringUtils.isBlank(fileKey)) {
        throw new IllegalStateException("群组 " + groupId + " 文件路径为空");
    }
    String mode = groupRule.getUpdateMode() != null ? groupRule.getUpdateMode() : "replace";
    Map<String, Object> result = groupService.importCsvToClickHouse(groupId, fileKey, mode);
    int count = ((Number) result.get("imported_count")).intValue();
    groupService.updateGroupCount(groupId, count);
    log.info("文件上传群组重新导入完成: groupId={}, count={}", groupId, count);
    return;
} else {
    throw new IllegalStateException("不支持的规则类型: " + groupRule.getType());
}
```

## 前端 - 重构 create-upload.vue（自动上传 + 文件状态展示 + 保存按钮控制）

**文件**: `profile-ui/src/views/group/create-upload.vue`

核心改造：

采用**两步方案**：
1. **上传阶段**：用户选择文件 → 前端 FileReader 本地预览 → 点"上传文件"上传到 MinIO → 上传成功后启用"保存群组"按钮
2. **保存阶段**：点"保存群组"

> **关键交互约束**：只有文件成功上传到服务器后，"保存群组"按钮才可点击。

### 7.1 交互流程（关键约束）

- **无单独"上传文件"按钮**，el-upload 组件，选择/拖拽文件后自动上传到 MinIO
- **上传中**：展示加载中图标（`uploading = true`），提示用户文件正在上传
- **上传成功后**：el-upload 文件列表区展示 "✅ 已完成 {文件名}" + 删除图标（如截图所示）
- 点击删除按钮：调用 `/group/cancel-upload` 接口删除 MinIO 文件，重置状态
- 底部操作区**只有"保存群组"按钮**，`uploaded = true` 时才可点击
- 提示文案："文件上传成功，请先保存群组，在群组列表中查看圈选结果"

### 7.2 底部操作区（只有保存按钮）

参考如下代码：
```html
<div class="page-actions">
  <el-button @click="goBack">取消</el-button>
  <el-button type="primary" @click="handleSave" :loading="saving" :disabled="!uploaded">
    保存群组
  </el-button>
</div>
```

状态流转：
- 初始："保存群组"禁用
- 选择文件 → `uploading = true`（显示加载中图标） → 上传成功 → `uploading = false, uploaded = true` → 展示文件名+删除按钮，"保存群组"启用
- 上传失败 → `uploading = false` → 提示错误，可重新选择文件
- 点击文件列表的删除按钮 → 删除 MinIO 文件 → `uploaded = false` → "保存群组"恢复禁用

### 7.4 上传回调逻辑

上传前校验从服务端转移到前端：
```ts
const uploading = ref(false)
const uploaded = ref(false)
const uploadedFileKey = ref('')
const selectedFileName = ref('')

// 上传前校验
const beforeUpload = (file: File) => {
  if (!file.name.toLowerCase().endsWith('.csv')) {
    ElMessage.error('仅支持 CSV 格式文件')
    return false
  }
  if (file.size > 200 * 1024 * 1024) {
    ElMessage.error('文件大小不能超过 200M')
    return false
  }
  uploading.value = true  // 开始上传，显示加载中图标
  return true
}

// 上传成功回调
const handleUploadSuccess = (response: any) => {
  uploading.value = false  // 上传完成，隐藏加载中
  uploadedFileKey.value = response.data  // MinIO object name
  selectedFileName.value = response.originalFilename || ''
  uploaded.value = true
}

// 上传失败
const handleUploadError = (error: any) => {
  uploading.value = false  // 上传完成，隐藏加载中
  ElMessage.error('文件上传失败')
  uploaded.value = false
}

// 点击删除按钮：删除 MinIO 文件，重置状态
const handleRemove = async () => {
  if (uploadedFileKey.value) {
    try {
      await groupApi.deleteUpload(uploadedFileKey.value)
    } catch (e) {
      console.warn('删除 MinIO 文件失败', e)
    }
  }
  uploaded.value = false
  uploadedFileKey.value = ''
  previewRows.value = []
  totalLines.value = 0
}
```

### 7.5 保存群组（上传成功后才可执行）

```ts
const handleSave = async () => {
  if (!uploaded.value || !uploadedFileKey.value) {
    ElMessage.warning('请先上传文件')
    return
  }
  saving.value = true
  try {
    const groupData = {
      group_name: basicForm.group_name,
      group_desc: basicForm.group_desc,
      entity_identifier_id: basicForm.entity_identifier_id,
      group_type: 2,
      trigger_type: 1,
      source_type: 2,
      group_rule: {
        type: 'upload',
        uuid_file_key: uploadedFileKey.value,
        file_list: [selectedFileName.value]
      }
    }
    const res = await groupApi.create(groupData)
    ElMessage.success('群组保存成功')
    router.push('/group')
  } catch (error: any) {
    ElMessage.error(error.response?.data?.message || '保存失败')
  } finally {
    saving.value = false
  }
}
```



## 验证计划

1. 选择/拖拽 CSV 文件 → 自动上传到 MinIO + 本地预览前 5 行
2. 上传成功后展示文件名称 + 删除按钮（如截图所示）
3. 上传成功后"保存群组"按钮**可点击**

5. 点击删除按钮 → 调用 MinIO 删除接口 → 重置状态
6. 点"保存群组" → 后端解析 CSV + 写入 ClickHouse + 创建群组记录
7. 保存成功后跳转群组列表，群组规模正确
8. ClickHouse 中 `profile_group_xxx` 表正确创建并写入数据
9. 下载模板 → 列名为 `entity_id`
10. 选择"追加"模式 → 编辑重新上传后新旧 ID 合并
11. 选择"覆盖"模式 → 编辑重新上传后旧数据被清空
12. 手动执行圈选 → GroupTask 从 MinIO CSV 重新导入成功
13. 取消页面 → 如已上传则自动删除 MinIO 文件（无孤儿文件）

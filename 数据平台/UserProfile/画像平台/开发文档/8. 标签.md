创建标签 #

在标签列表右侧点击新建标签按钮“+”，在弹出的弹框中选择标签的创建方式。当前提供「数据源导入、自定义标签、SQL、上传文件」4种用户标签的创建方式：
数据源导入：从数据源导入画像平台时，通过为字段配置标签别名的方式给用户打标签，后续使用标签时，实质上是在使用对应的字段。
自定义标签：可基于四则运算、简单函数实现标签的计算。
SQL：可直接使用SQL语言进行标签的新建。
上传文件：上传包含实体ID(例如用户)的文件，可直接创建标签。

数据源导入方式功能开发：
- 点击数据源导入的立即创建，跳转标签创建页面，分为两步操作
- 第一步：标签信息
  - 包含标签名称、标签类目(下拉选择，默认未分类，根据层级选择，传递后端是最小层级类目Id)、标签描述、标签实体标识(下拉选择实体标识,格式为 entity_id>entity_identifier，传给后端是entity_identifier)
  - 标签类型(label_type,默认为属性标签)、数据类型(对应data_type，默认为文本型)、数据分布类型(dist_type，默认枚举)、组织类型(organize_type，默认为单值)、加工类型(produce_type，默认事实标签)、时效性类型(time_type,默认离线标签)
    - 下拉选择通过 /label/config 后端接口获取
- 第二步：标签来源,可以选择跳过
  - 如果选择跳过，标签状态(labelStatus)设置为待上架(0),否则设置为在线(1)
  - 实体标识选择
    - 下拉选项格式：实体类型>实体标识（如 用户>uid）
    - 通过 /entity/identifier/list 接口获取数据
    - 点击保存时也需要把实体标识 entity_identifier 传给后端。
  - 数据集名称(下拉选择只能选择标签数据集)
  - 数据集字段(下拉选择)

最后点击保存或者取消完成数据源导入方式的创建。



标签类目逻辑
- 一级类目 - 页面加载时自动加载所有一级类目（category_level: 1）；清空二级、三级选择
- 二级类目 - 选择一级类目后，根据 parent_category_id 动态加载对应的二级类目；清空三级选择
- 三级类目 - 选择二级类目后，根据 parent_category_id 动态加载对应的三级类目
- 保存时 - 保存时根据选择的最深层级确定 category_id，即如果选了三级就传三级ID，选了二级就传二级ID，选了一级就传一级ID）

优化标签列表页：标签类型通过 /label/config 对应获取对应ID的中文名称；添加实体标识列，格式：实体类型>实体标识（如 用户>uid）；

标签宽表协议：
```sql
CREATE TABLE IF NOT EXISTS `user_profile_label`(
  `user_id` bigint COMMENT '用户ID',
  `gender` string COMMENT '性别',
  `brand` string COMMENT '品牌',
  `prov` string COMMENT '常驻省份',
  ...
)
COMMENT '画像标签宽表'
PARTITIONED BY (`dt` string COMMENT '日期-yyyyMMdd');
```

> 标签ID -> 标签宽表数据集 -> 物理表字段

标签Bitmap表协议：
```sql
CREATE TABLE IF NOT EXISTS `user_profile_bitmap_label`(
  `bucket_id` bigint COMMENT '标签分桶',
  `label_name` string COMMENT '标签名称：例如，性别',
  `label_value` string COMMENT '标签值：例如，男',
  `label_bitmap` binary COMMENT '标签用户Bitmap'
)
COMMENT '画像标签Bitmap表'
PARTITIONED BY (`dt` string COMMENT '日期-yyyyMMdd');
```
> 标签ID -> 标签Bitmap数据集 -> label_name = 'xxx'

标签BSI表协议：
```sql
CREATE TABLE IF NOT EXISTS `user_profile_bsi_label`(
  `bucket_id` bigint COMMENT '标签分桶',
```

```sql
CREATE TABLE IF NOT EXISTS `profile_meta_label`(
    `id` BIGINT UNSIGNED AUTO_INCREMENT COMMENT '自增ID',

    `label_category_id` VARCHAR(100) NOT NULL COMMENT '标签类目ID',
    `label_data_type` INT NOT NULL DEFAULT 1 COMMENT '标签数据类型: 1-文本型,2-数值型,3-时间型',
    `label_dist_type` INT NOT NULL DEFAULT 1 COMMENT '标签数据分布类型: 1-枚举,2-非枚举',
    `label_organize_type` INT NOT NULL DEFAULT 1 COMMENT '标签组织类型: 1-单值,2-多值,3-KV,4-KKV',
    `label_produce_type` INT DEFAULT 0 COMMENT '标签加工类型: 0-未知,1-事实标签,2-统计标签,3-预测标签',
    `label_time_type` INT DEFAULT 0 COMMENT '标签时效性类型: 0-未知,1-离线标签,2-实时标签',
    `entity_identifier_id` VARCHAR(500) COMMENT '标签实体标识ID',
    `source_type` INT NOT NULL DEFAULT 1 COMMENT '创建方式: 1-系统内置,2-数据集导入,3-文件上传,4-四则运算,5-SQL计算,6-自定义规则,7-API导入,8-数据表导入',
    `config` VARCHAR(500) NOT NULL COMMENT '标签计算规则,不同创建方式不同规则',
    `is_office` INT NOT NULL DEFAULT 0 COMMENT '是否官方认证:0-否,1-是',
    `owner` VARCHAR(100) NOT NULL COMMENT '标签负责人',
    `creator` VARCHAR(100) NOT NULL COMMENT '标签创建者',
    `modifier` VARCHAR(100) NOT NULL COMMENT '标签最后修改者',
    `gmt_create` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '标签创建时间',
```
